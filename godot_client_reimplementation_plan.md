# Godot 客户端重构计划

本文档面向“使用 Godot 重新实现 BrowserQuest 原项目客户端”的目标，重点整理需要迁移的功能模块、建议的实现顺序与推荐的开发步骤。

## 1. 目标与范围

目标：用 Godot 重建原有浏览器客户端的核心体验，重点实现以下能力：

- 进入游戏、角色创建、游戏启动
- 地图显示与摄像机控制
- 玩家移动、交互、攻击
- 怪物、NPC、物品、宝箱等实体表现
- 聊天、UI、血条、装备栏、成就等界面
- 与服务端的消息同步
- 音效、动画、特效等表现层

说明：这里的计划主要面向“客户端重建”，不包含服务端逻辑重写。

---

## 2. 客户端需要实现的功能清单

### 2.1 启动与界面系统

- 启动场景与主菜单
- 角色创建/读取角色信息
- 游戏进入动画与过渡效果
- 聊天窗口显示/隐藏
- 帮助、成就、关于、信用信息页面
- 玩家数量显示与人口信息
- 断线/重连提示

### 2.2 网络与协议层

- 与服务端建立连接
- 处理连接成功、断线、超时等状态
- 解析服务端消息协议
- 将消息分发到对应系统模块
- 支持玩家移动、攻击、掉落、聊天、装备变化等消息处理

### 2.3 地图系统

- 地图资源加载
- 地图瓦片渲染
- 地图视野与摄像机跟随
- 地图层次（地面、物体、前景）
- 碰撞检测与路径可行性判断

### 2.4 实体系统

- 玩家实体
- 怪物实体
- NPC 实体
- 物品实体
- 宝箱实体
- 实体状态管理（空闲、移动、攻击、死亡等）
- 实体动画与方向控制

### 2.5 输入与交互

- 鼠标点击移动
- 鼠标选择目标
- 攻击、拾取、交互操作
- 键盘输入与移动控制
- 移动端/触控适配（如果需要）

### 2.6 战斗系统

- 攻击逻辑
- 伤害计算与显示
- 生命值变化显示
- 死亡与重生流程
- 防御/无敌/受伤反馈

### 2.7 UI 与信息展示

- 血条、经验条、状态条
- 装备栏
- 物品栏与背包信息
- 成就系统界面
- 提示信息、浮动文本、消息气泡
- 角色名称与头像显示

### 2.8 音效与特效

- 音效播放
- 攻击、受伤、拾取、死亡音效
- 特效（爆炸、打击、粒子、闪烁）

### 2.9 本地数据与持久化

- 玩家昵称、角色外观、装备保存
- 成就进度保存
- 本地设置保存

## 3. 推荐的实现顺序

### 阶段 1：项目搭建与基础框架

目标：先搭建一个能跑起来的 Godot 客户端骨架。

建议步骤：

1. 新建 Godot 项目，并建立基础目录结构
   - scenes/
   - scripts/
   - resources/
   - data/
2. 实现主场景与启动流程
   - 登录/开始界面
   - 游戏主界面容器
3. 建立基础资源管理器
   - 纹理、音效、字体、地图资源加载器
4. 建立全局单例
   - GameManager
   - NetworkManager
   - AudioManager
   - SaveManager

输出结果：
- 能显示主界面
- 能切换场景
- 能加载基础资源

---

### 阶段 2：网络层与消息协议

目标：先让客户端能够和服务端通信。

建议步骤：

1. 实现 WebSocket 或现有服务端兼容的通信层
2. 定义消息结构与消息处理器
3. 先支持最小可用协议：
   - 连接成功
   - 欢迎消息
   - 玩家移动
   - 玩家生成/销毁
4. 建立事件总线或消息分发系统

输出结果：
- 客户端可以收到服务端消息
- 可以把消息转发给对应模块

---

### 阶段 3：地图与摄像机

目标：把世界显示出来。

建议步骤：

1. 实现地图资源加载
2. 绘制二维地图瓦片
3. 实现相机跟随玩家
4. 支持缩放与视野范围
5. 处理地图层级与遮挡表现

输出结果：
- 能看到地图和玩家角色
- 摄像机能跟随角色移动

---

### 阶段 4：实体系统与角色表现

目标：让世界中出现玩家、NPC、怪物、物品等实体。

建议步骤：

1. 定义实体基类
2. 实现玩家角色节点
3. 实现怪物/NPC/物品/宝箱节点
4. 为实体添加动画状态机
5. 让实体根据服务端消息更新位置、方向、状态

输出结果：
- 玩家能出现在地图上
- 其它实体能被生成和销毁

---

### 阶段 5：输入与基础交互

目标：让玩家能实际操作世界。

建议步骤：

1. 实现点击移动
2. 实现攻击目标选择
3. 实现拾取物品
4. 实现交互 NPC / 宝箱
5. 处理鼠标悬停与高亮显示

输出结果：
- 可以移动角色
- 可以攻击或互动对象

---

### 阶段 6：战斗与游戏核心系统

目标：补齐核心玩法。

建议步骤：

1. 实现生命值与伤害显示
2. 处理攻击动画与命中反馈
3. 实现死亡和重生
4. 支持装备变化与显示
5. 处理无敌、受伤、闪烁等状态

输出结果：
- 能完成基本战斗循环
- 玩家状态变化可视化

---

### 阶段 7：UI 与信息系统

目标：完善用户体验。

建议步骤：

1. 实现血条、状态条、装备栏
2. 实现聊天框与消息气泡
3. 实现成就界面与提示系统
4. 实现玩家数量与场景信息展示
5. 增加操作提示与引导内容

输出结果：
- 具备与原客户端相近的操作界面

---

### 阶段 8：音效、特效与视觉优化

目标：增强体验和沉浸感。

建议步骤：

1. 接入音效系统
2. 添加攻击、受伤、拾取、死亡音效
3. 加入粒子特效与打击反馈
4. 优化角色动画与摄像机表现
5. 补齐阴影、光照、层次感等视觉效果

输出结果：
- 游戏视觉和声音体验更完整

---

### 阶段 9：持久化与完善

目标：让客户端更接近原有产品形态。

建议步骤：

1. 实现本地存档
2. 保存角色外观、装备、成就进度
3. 实现设置项
4. 处理断线重连与错误提示
5. 进行性能优化与兼容性测试

输出结果：
- 具备可持续使用的客户端基础版本

## 4. 推荐的模块组织方式

可以按下面方式组织 Godot 项目：

- scenes/
  - main_menu.tscn
  - game.tscn
  - ui/
  - entities/
- scripts/
  - global/GameManager.gd
  - global/NetworkManager.gd
  - global/AudioManager.gd
  - global/SaveManager.gd
  - entities/Player.gd
  - entities/Npc.gd
  - entities/Mob.gd
  - entities/Item.gd
  - ui/HUD.gd
  - world/MapManager.gd
  - world/CameraController.gd
- resources/
  - tilesets/
  - sprites/
  - audio/
  - data/

---

## 5. 建议的 MVP 版本

如果想先快速验证方向，建议先实现一个最小可玩版本（MVP）：

- 主菜单
- 角色进入地图
- 玩家移动
- 服务器消息同步
- 基础攻击与血条
- 简单 UI

这个版本可以先上线测试交互逻辑，再逐步扩展成完整客户端。

---

## 6. 实施建议

1. 先做“能显示世界”的版本，再补“能互动”的功能。
2. 网络层与游戏逻辑尽量解耦，便于后续维护。
3. 先实现通用实体基类，再逐步派生玩家、怪物、NPC。
4. 先确保核心循环稳定，再做视觉细节和特效。
5. 每个阶段都应该有一个可验证的目标，避免一次性做太多。
